Práctica: Control de versiones con Git y GitHub
Objetivos de la práctica
●	Inicializar y configurar un repositorio Git local.
●	Gestionar versiones mediante commits y ramas.
●	Resolver conflictos y fusionar ramas.
●	Conectar el repositorio local con GitHub y sincronizarlo.
●	Colaborar con otros compañeros mediante forks.
●	Documentar correctamente el proyecto con un README.md
Contenidos evaluados
●	Conceptos básicos de Git (repositorio, commit, branch, merge).
●	Comandos principales (git init, add, commit, status, log, branch, merge, push, pull, clone, remote, stash).
●	Flujo de trabajo con GitHub (repositorio remoto).
●	Resolución de conflictos.
●	Buenas prácticas en la documentación y mensajes de commit.
Descripción de la práctica
Vas a desarrollar un pequeño proyecto en equipo (2-3 personas) usando Git y GitHub. El proyecto consistirá en crear una miniaplicación o script sencillo en el lenguaje Java, (por ejemplo, un conversor de temperaturas, un contador, una calculadora, o un gestor de tareas básico).
Etapas del trabajo
1. Configuración inicial
●	Crea una carpeta de proyecto en tu equipo.
●	Inicializa un repositorio Git..
●	Configura tu nombre y correo.
●	Crea un archivo .gitignore apropiado para tu lenguaje.
2. Primer commit
●	Crea un archivo principal y haz un primer commit.
●	Añade un archivo README.md con título, descripción, integrantes e instrucciones de uso.
●	Uso de tags para marcar versiones.
3. Creación de ramas
●	Cada integrante debe crear su propia rama y trabajar en una funcionalidad diferente.
●	Realiza al menos 3 commits significativos por persona.
4. Fusión de ramas
●	Fusiona las ramas en la principal usando merge o pull request.
●	Documenta en el README las funcionalidades añadidas.
●	Resuelve conflictos si los hay.
5. Repositorio remoto
●	Crea un repositorio en GitHub.
●	Conéctalo con el local usando git remote add origin.
●	Sube tu trabajo con git push -u origin main.
6. Colaboración
●	Un compañero debe hacer un fork de vuestro repositorio y enviar un pull request.
●	Acepta el pull request tras revisarlo.
Entrega
Debes entregar en el aula virtual:
●	Enlace al repositorio GitHub.
●	Capturas de pantalla de la consola con los comandos usados.
●	Captura de la resolución de un conflicto.
●	Informe breve en PDF explicando el flujo de trabajo seguido.

Rúbrica de evaluación
Criterio	Ponderación	Descripción
Inicialización y configuración del repositorio	10%	Repositorio correctamente configurado con .gitignore y README.md
Uso de commits y ramas	25%	Commits claros y frecuentes, ramas correctamente creadas y fusionadas
Resolución de conflictos	15%	Conflictos gestionados correctamente, sin pérdida de información
Conexión con GitHub	20%	Repositorio remoto funcional y sincronizado
Colaboración (fork + pull request)	15%	Participación activa en el flujo colaborativo
Documentación	15%	README completo y de calidad, buena presentación del informe
